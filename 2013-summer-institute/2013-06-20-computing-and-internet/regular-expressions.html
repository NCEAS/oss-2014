Regular expressions

What is a regular expression?

Notation for describing a set of character strings

Examples
foo matches foo
[fb]oo matches foo and boo
fo* matches f, fo, foo, ...

Why is this useful?

Matching patterns to text lets you...

  • Find data
  • Filter data
  • Validate data
  • Sanitize data

Places you might encounter regexes

Editors
vim, emacs, TextMate, Notepad++, ...
Languages
e.g. in R: grep, sub, gsub, regexpr
Utilities
grep, sed, awk, find, locate, ...

Regular expression: patterns

  • Characters
    • a, Z, 5, @
  • Character sets
    • [aeiou] some set of characters
    • [^aeiou] ...or not those characters!
    • [a-zA-Z] ...or a range
    • [[:alpha:]] ...or a POSIX character class
  • Any character
    • .

Regular expression: patterns

  • Anchors
    • ^stuff at start of line
    • stuff$ at end of line
    • \<stuff\> as its own word
  • Quantifiers
    • ? zero or one
    • * zero or more (any)
    • + one or more (at least one)
    • {n} exactly n
    • {,n} no more than n
    • {n,} at least n
  • Alternation
    • foo|bar ...one of two possible patterns

Escaping

  • What does a+ match?
    • is it the literal {a+}?
    • or is it {a, aa, aaa, ...}?
    • answer: it depends!
  • Here are the usual metacharacters:
    • { } [ ] ( ) ^ $ . | * + ? \
  • When in doubt, test your pattern!
    • try both a+ and a\+
